Arduino Electrónica Attiny

- Resumen -

El ATtiny es una familia de microcontroladores desarrollada por Atmel (ahora parte de Microchip Technology) que se caracteriza por su pequeño tamaño, bajo costo y consumo reducido de energía. Forma parte de la arquitectura AVR, al igual que otros microcontroladores de Atmel, pero está diseñado específicamente para aplicaciones donde el espacio es limitado y no se necesitan muchas entradas/salidas o características avanzadas.

Proyectos DIY y sistemas embebidos que requieren microcontroladores pequeños y eficientes. Aplicaciones portátiles, dispositivos wearables, pequeños robots y sistemas de automatización simples. A pesar de ser limitados en comparación con microcontroladores más grandes, los ATtiny son una excelente opción para proyectos sencillos o donde el bajo consumo y tamaño son prioritarios.

- Introducción -

En el desarrollo de sistemas embebidos, la elección del microcontrolador adecuado es fundamental para optimizar el espacio, el consumo de energía y los recursos disponibles. El ATtiny85, un microcontrolador compacto y eficiente de 8 bits, es una opción popular para proyectos pequeños donde se requiere un hardware reducido. Sin embargo, debido a su tamaño, el ATtiny85 no incluye una interfaz USB directa para su programación. Para solventar esta limitación, se puede utilizar un Arduino Uno como programador In-System (ISP), permitiendo cargar código directamente al ATtiny85.

El uso del Arduino Uno como ISP es una técnica ampliamente utilizada para programar microcontroladores pequeños como el ATtiny85 sin la necesidad de hardware especializado. Esta práctica no solo simplifica el proceso de programación, sino que también permite integrar el ATtiny85 en aplicaciones que requieren un menor consumo energético y espacio reducido, sin sacrificar las funcionalidades básicas de control.

En este reporte se describirá el proceso de configuración de un Arduino Uno como ISP y los pasos necesarios para programar un ATtiny85, abordando tanto la teoría detrás de la programación ISP como la aplicación práctica de este método en proyectos reales.

- Desarrollo -

Preparación del Arduino Uno como ISP El primer paso es configurar el Arduino Uno para que funcione como un programador ISP (In-System Programmer). Esto se hace cargando el ejemplo "ArduinoISP" desde el software Arduino IDE. El procedimiento es el siguiente:

1. Abrir Arduino IDE: Desde el menú, ir a Archivo > Ejemplos > 11.ArduinoISP > ArduinoISP.

2. Seleccionar la placa y puerto del Arduino Uno: Ir a Herramientas > Placa y seleccionar Arduino Uno. Luego, en Puerto, elegir el puerto al que está conectado el Arduino.

3. Cargar el código: Presionar el botón "Cargar" para transferir el código de ArduinoISP al Arduino Uno. Esto lo convierte en un programador.

Conexión del Arduino Uno al ATtiny85 Una vez que el Arduino está preparado como programador ISP, es necesario realizar las conexiones entre el Arduino y el ATtiny85. Las conexiones deben ser precisas, utilizando los pines adecuados de cada dispositivo. Las conexiones son las siguientes:

Pin 10 (Arduino) → Pin 1 (Reset, ATtiny85) Pin 11 (Arduino) → Pin 5 (MOSI, ATtiny85) Pin 12 (Arduino) → Pin 6 (MISO, ATtiny85) Pin 13 (Arduino) → Pin 7 (SCK, ATtiny85) 5V (Arduino) → Pin 8 (VCC, ATtiny85) GND (Arduino) → Pin 4 (GND, ATtiny85)

Instalación del soporte para ATtiny85 en Arduino IDE El ATtiny85 no está incluido por defecto en las placas soportadas por Arduino IDE, por lo que se deben seguir los siguientes pasos para agregarlo:

1. Abrir Arduino IDE e ir a Archivo > Preferencias.

2. Luego, ir a Herramientas > Placa > Gestor de tarjetas y buscar "attiny". Instalar el soporte para el ATtiny.

Configuración de parámetros de programación Con las conexiones y el soporte para ATtiny85 instalados, es necesario configurar los parámetros correctos en el Arduino IDE: En Herramientas > Placa, seleccionar ATtiny25/45/85. En Procesador, seleccionar ATtiny85. En Frecuencia, seleccionar la frecuencia del reloj deseada, por ejemplo, 8 MHz (internal). En Programador, seleccionar Arduino as ISP.

Quemar el bootloader (opcional) Para asegurarse de que el ATtiny85 esté correctamente configurado para funcionar con la frecuencia interna seleccionada, es recomendable "quemar" el bootloader:

1. En Herramientas, seleccionar la opción Quemar bootloader.

2. Este proceso inicializa el microcontrolador para que funcione con la frecuencia seleccionada y se asegura de que esté listo para la programación.

Finalmente, el ATtiny85 está listo para recibir un programa. A modo de ejemplo, se puede cargar un código sencillo como el parpadeo de un LED:

void setup() { pinMode(0, OUTPUT); // Pin 0 (físico pin 5 en ATtiny85) } void loop() { digitalWrite(0, HIGH); delay(1000); digitalWrite(0, LOW); delay(1000); }

Escribir el código en el Arduino IDE.

Seleccionar Subir usando programador en el menú de Sketch para cargar el código en el ATtiny85.

Con estos pasos, el ATtiny85 queda programado utilizando un Arduino Uno como ISP. Este método es práctico para integrar microcontroladores pequeños en proyectos sin necesidad de hardware adicional, permitiendo maximizar las ventajas de los sistemas embebidos compactos.

- Conclusiones -

Durante la práctica, se demostró cómo las conexiones correctas entre ambos dispositivos, junto con una configuración adecuada del entorno de programación en Arduino IDE, permiten cargar código en el ATtiny85 de manera sencilla. Este enfoque es especialmente útil en proyectos donde se necesita optimizar el espacio y el consumo de energía, ya que el ATtiny85 ofrece un tamaño reducido y características básicas suficientes para controlar elementos como LEDs, sensores o motores pequeños.

Siendo una herramienta valiosa para proyectos de sistemas embebidos y controladores simples, facilitando el desarrollo de soluciones compactas y energéticamente eficientes.
